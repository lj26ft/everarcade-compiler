Honest playability finding

The arena template is not actually playable as a hosted local EverArcade game. It is a creator-template script with metadata and a `start()` function. The creator SDK can simulate create/build/test/deploy/publish JSON artifacts, but those artifacts are not accepted by `runtime/everarcade-runtime`, whose package loader expects `manifest.json`, a wasm payload, hash, signature string, and world metadata. The runtime crate could not be run in this environment due to incomplete vendored dependencies.

Final classification rationale

- Not `Playable`: no interactive player loop, authoritative runtime execution, real session join, renderer, or gateway flow was proven.
- Not fully `Broken`: the template CLI and game script do run.
- Better than `Script-only` only in the sense that build/deploy JSON artifacts are produced.
- Therefore the end-to-end local game launch is **Simulated**.
