# Arena Vanguard HotPocket Cluster Package

Deploy these exact bytes to each HotPocket node. Each node must use its own working directory and persistence directory; do not share `state/` between nodes.

## Preflight

Run `sha256sum` for `contract/hotpocket-adapter.mjs`, `runtime/runtime.mjs`, and `genesis/genesis.json` on every node and compare with `reports/live-hotpocket-cluster/preflight.json` generated by the validation harness.

## Node launch

From an independent node directory containing this package:

```bash
npm install hotpocket-nodejs-contract
node contract/hotpocket-adapter.mjs
```

Submit player inputs with a HotPocket client as JSON payloads:

```json
{"action":"join","player":"player-1"}
```

The adapter consumes `ctx.users`, uses `ctx.lclSeqNo` as the deterministic tick source, persists state and journal under local `state/`, and emits receipts/commitments to users.


## Live cluster evidence harness

From the repository root, generate the required milestone artifacts with:

```bash
bash scripts/validate_hotpocket_live_cluster.sh
```

The harness stages three isolated node directories under `reports/hotpocket-live/node-a`, `node-b`, and `node-c`, drives the adapter only through HotPocket-style `ctx.users` input surfaces, records `ctx.lclSeqNo` and `ctx.npl`, compares `state_root`, `receipt_root`, `world_hash`, and `continuity_root` after every round, replays each node journal, and writes `reports/hotpocket-live/cluster-report.md`.

To inspect already-captured rounds for the first divergent commitment, run:

```bash
bash scripts/detect-live-divergence.sh reports/hotpocket-live
```
