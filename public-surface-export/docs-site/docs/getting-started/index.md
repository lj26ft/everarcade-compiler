# Getting Started

Use this path to understand and run EverArcade quickly.

1. Read the project summary in the public documentation and the [Open Source Readiness](/docs/open-source/open-source-readiness) note while the repository is prepared for release.
2. Review the capability and maturity pages to distinguish proven, alpha, experimental, scaffold, and planned areas.
3. Follow the existing [30-minute developer journey](../onboarding/30-minute-developer-journey.md).
4. Use the [Creator SDK quick start](../creator-sdk/quick-start.md) and [runtime package bridge](../creator-sdk/runtime-package-bridge.md) for local packaging.
5. Consult repository maps and code references after the open-source repository is released.

The fastest validated path remains local creator onboarding and local runtime proof. Do not infer production, public-testnet, settlement, marketplace, renderer, or federation readiness from local PASS reports.
