# v0.1.0 Runtime Surface Audit

Classification key:
- **Production-ish**: used by CLI/runtime flows and exercised by release scripts.
- **Prototype**: partially integrated, works for local experiments.
- **Scaffold**: structural modules present with low runtime integration.
- **Placeholder**: stubs/todo-grade behavior.
- **Deprecated**: superseded or intentionally non-primary.

| Component | Status | What works | What is missing | v0.1.0 required? |
|---|---|---|---|---|
| `everarcade-host/src/main.rs` + core CLI (`init/run/verify/status/debug/doctor`) | Production-ish | End-to-end local operator loop, deterministic fixture run/verify, state introspection. | Full production hardening and richer error taxonomy. | Yes |
| `everarcade-host/src/fixture/` | Production-ish | Deterministic package generation used by smoke/release validation. | More realistic game-level fixtures. | Yes |
| `everarcade-host/src/state_folder/` | Production-ish | State layout, manifest handling, validation/reporting used by CLI. | Long-horizon compaction/retention policy. | Yes |
| `everarcade-host/src/verify/` | Production-ish | Package/receipt/checkpoint/anchor verification in local flow. | External trust root integration at production level. | Yes |
| `everarcade-host/src/operator/` | Prototype | Operator config/profile wiring and environment structure. | Full policy + secret management lifecycle. | Yes (minimum) |
| `everarcade-host/src/network/` + serve/sync CLI | Prototype | Basic TCP sync handshake for local peer experiments. | Authenticated multi-peer protocol/security hardening. | No |
| `everarcade-host/src/*` distributed/governance/civilization/treaty/etc families | Scaffold | Namespace and model skeletons compile. | End-to-end orchestrated runtime semantics. | No |
| `execution-core/src/execute.rs` + ABI boundary | Production-ish | Deterministic VM input/output execution path and serialization boundary. | Full WASM game contract ABI and limits policy finalization. | Yes |
| `execution-core/src/*` economics/domain/federation/etc families | Scaffold | Compilable module surface for future architecture. | Runtime-connected implementations and validation. | No |
| `scripts/install.sh` + `scripts/uninstall.sh` | Production-ish | Prefix/bin-dir install and clean uninstall from tarball. | Cross-distro packaging variants. | Yes |
| `scripts/install_smoke.sh` | Production-ish | Automated extract/install/run/verify/status/uninstall smoke. | Broader matrix coverage. | Yes |
| `scripts/release_validate.sh` | Production-ish | Offline-lean validation flow with build/test/smoke/install checks. | CI artifact attestation/signing integration. | Yes |
| `docs/operator-release.md`, `docs/linux-vm-operator-quickstart.md` | Prototype | Operator setup and release flow guidance. | Tighter alignment with stdout protocol + deployment bundle docs. | Yes |
| Broad architecture docs under `docs/` (distributed, XRPL, Evernode, governance) | Scaffold | Directional design context for roadmap and experiments. | Must avoid overpromising production readiness. | No |
