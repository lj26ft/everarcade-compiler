# Developers

Developers build worlds and capabilities for EverArcade.

## What you can build

- A `world.evr` package for a playable world.
- A World Git contribution, fork, migration, or restore workflow.
- A verified replay or proof verifier.
- A capability for the Capability Marketplace.
- A treasury policy or execution helper.
- A portal, client, or operator tool that makes verified worlds easier to use.

## Build path

1. Create a world package.
2. Add registry metadata and contributor opportunities.
3. Generate replay evidence.
4. Add restore and migration fixtures.
5. Publish to the world portal.

[Read capability docs](./capabilities.md) or [apply for Founding Worlds](../founding-worlds.md).
