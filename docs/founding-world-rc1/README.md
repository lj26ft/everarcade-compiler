# Frontier Settlement RC1 Founding World

Frontier Settlement RC1 is the first complete persistent EverArcade reference world. It is generated by the World Factory from the Frontier Settlement blueprint, packaged as `world.evr`, executed by the small deterministic runtime, replay verified, remotely verified against a live HTTP host, release-attested, and exported for portal discovery.

## Public entry points

- Portal record: `public-surface-export/world-portal/src/data/registryFixtures.ts` (`frontier-settlement`).
- World Factory project: `examples/world-factory/frontier-settlement/`.
- Public proof bundle: `exports/founding-world-rc1-proof-bundle/`.
- Operator guide: `docs/founding-world-rc1/operator-guide.md`.
- Creator guide: `docs/founding-world-rc1/creator-guide.md`.

## What the world exercises

| Requirement | RC1 evidence |
| --- | --- |
| Generated through World Factory | `world-blueprint.json`, `world-contract-plan.json`, and generated `out/world.evr/` package. |
| Frontier Settlement template | `world_type` is `frontier-settlement`. |
| `inventory.transfer` | Runtime journal ticks 1 and 4 transfer timber into the settlement store. |
| `market.trade` | Runtime journal ticks 2 and 5 trade ore for coins. |
| `governance.vote` | Runtime journal ticks 3 and 6 cast council votes on `proposal-001`. |
| Operator identity | Release attestation uses attester `frontier-settlement-operator-rc1`. |
| Package verification | `world factory verify` passes against `out/world.evr/`. |
| Replay verification | `world factory replay` writes `out/runtime/world-factory-runtime-report.json` with `replay_status: PASS`. |
| Remote verification | `world factory proof --url http://127.0.0.1:8787` writes `live-deployment-proof.json` with `remote_verification: PASS`. |
| Release attestation | `out/release/world-release-attestation.json` and `trusted-public-key.txt`. |
| Deployment proof | `out/deploy/deployment-report.json`, `deployment-manifest.json`, and `live-deployment-proof.json`. |
| Runs on a live host | The RC1 proof was collected from the live World Factory HTTP host at `http://127.0.0.1:8787` during validation. |
| Portal page | Exported portal registry fixture presents this world to new users. |
| Operator workspace | `out/deploy/`, `out/runtime/`, and the operator guide define the workspace. |
| Contributor opportunities | Portal record and creator guide list economy design, governance scenario, verification review, and settlement content opportunities. |

## New user path

1. Visit the portal and select **Frontier Settlement RC1**.
2. Open the proof bundle linked from the portal record.
3. Verify package, replay, remote proof, and release attestation with the commands in the operator guide.
4. Join through the operator endpoint advertised by the current deployment manifest.
5. Read the creator guide for world rules, contribution lanes, and governance process without reading compiler internals.
