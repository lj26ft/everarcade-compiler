# EverArcade Risk Register

Canonical assessment date: 2026-06-03

| Risk Name | Description | Impact | Likelihood | Mitigation | Owner | Status |
| --- | --- | --- | --- | --- | --- | --- |
| Determinism Regression | Runtime, ABI, hashing, fuel, or serialization changes cause identical inputs to produce different outputs. | Critical: invalid receipts/replays and loss of trust. | Medium | Freeze deterministic engine config; add artifact-level replay equivalence gate; require canonical hash fixtures for ABI and state roots. | Runtime/Core | Open |
| Replay Divergence | Replayed journal or frame stream does not reproduce the committed state root. | Critical: recovery and independent verification fail. | Medium | Gate every release on replay from persisted artifacts; reject gaps/reordering; publish replay root in health/status. | Runtime/Core | Open |
| Checkpoint Corruption | Checkpoint files are missing, partially written, stale, or inconsistent with journal/receipts. | Critical: world cannot recover after crash. | Medium | Add fsync/atomic directory updates; checkpoint validation; scheduled restore rehearsals; backup verified copies. | Persistence | Open |
| Journal Chain Break | Journal sequence or previous-entry hash chain breaks after crash, manual edit, or storage fault. | High: replay cannot advance safely. | Medium | Maintain existing sequence/hash checks; add corruption matrix tests and operator repair/quarantine workflow. | Persistence | Open |
| Federation Split-Brain | Multiple peers continue independently with incompatible roots/topology. | Critical for federated worlds; not acceptable for public federation. | High if federation is enabled publicly | Keep federation out of v0.1 production scope; require quorum/topology protocol, peer auth, partition tests, and convergence proof. | Federation | Open |
| Peer Recovery Failure | A failed/stale peer cannot resync checkpoints/replay or rejoins with stale state. | High | High in live distributed deployment until proven | Cross-machine restart/rejoin tests; stale-peer quarantine; checkpoint catch-up and replay equivalence gate. | Federation/Ops | Open |
| Renderer Desynchronization | Renderer misses frames or accepts invalid projection data, causing incorrect client view. | Medium to High depending on product promise. | High | Keep renderer non-authoritative; add projection stream tests, missed-frame catch-up, archive validation, and observer recovery tests. | Renderer | Open |
| Renderer Authority Confusion | Client/renderer code path appears able to mutate authoritative state or users misunderstand boundary. | High security/product risk. | Medium | Enforce command-only input boundary; document renderer as non-authoritative; add tests proving renderer mutations are rejected. | Runtime/Docs | Open |
| Lease Recovery Failure | Evernode lease restart, disk pressure, or provider lifecycle event loses world state. | Critical for Evernode v0.1. | High until live trial | Run live lease trial; map storage paths; validate restart/recover/rollback; monitor disk/replay growth. | Evernode/Ops | Open |
| Resource Exhaustion | Replay, checkpoints, assets, or packages exhaust disk/memory/CPU/fuel quotas. | High | Medium | Enforce quotas; add retention/compaction; alert on storage forecasts; run low-resource validation. | Runtime/Ops | Open |
| XRPL Anchor Drift | Local anchor intent and ledger-confirmed state differ, or placeholder tx ids are mistaken for settlement. | High if settlement is enabled. | Medium | Keep live settlement disabled; implement idempotent anchor queue, ledger verification, finality policy, and reconciliation. | Settlement | Open |
| Key/Secret Exposure | Future live XRPL/Xahau or operator keys are logged, committed, or stored insecurely. | Critical | Medium | Define secret storage/redaction policy; never include secrets in deterministic state or logs; add scans and docs. | Security/Ops | Open |
| Operator Error | Operator runs wrong script, misreads scaffold status, deletes state, or performs unsafe upgrade. | High | High | Reduce commands to canonical workflows; label scaffold commands; add `doctor`, dry-run, backups, and rollback confirmation. | Ops/CLI | Open |
| False Validation Confidence | Generated reports or placeholder scripts imply readiness without executing real behavior. | High roadmap risk. | High | Treat only code/tests/runtime behavior as evidence; label synthetic reports; make release gates fail on placeholders. | Release | Open |
| Public API Instability | Broad `execution-core` module surface becomes de facto public API and breaks developers. | Medium | High | Publish stable facade; document semver policy; hide/internalize experimental modules. | SDK/Core | Open |
| Package Supply Chain Risk | Untrusted packages, vendored dependencies, or generated artifacts are not verifiably safe/reproducible. | High | Medium | Signed artifacts, checksums, SBOM/dependency review, sandbox policy, package validator, and offline/frozen build gates. | Release/Security | Open |
| Backup Gap | Operators have no automated backup or verified restore cadence. | Critical after paid launch. | High | Implement backup command, retention policy, restore drill, and backup integrity report. | Ops/Persistence | Open |
| Observability Blind Spot | Runtime fails without metrics/logs/alerts that operators can act on. | High | High | Add health/metrics/log export, dashboards, alert routes, and support bundle. | Ops | Open |
| Upgrade/Rollback Failure | Runtime upgrade changes schema/protocol and cannot be rolled back safely. | High | Medium | Version schemas; perform pre-upgrade backup; run migration dry-run; validate rollback gate. | Release/Ops | Open |
| Scope Creep Before v0.1 | Advanced domains delay the deployable deterministic world runtime. | Medium | High | Freeze v0.1 to single-world deploy/recover path; mark federation/renderer/XRPL as post-v0.1 unless separately proven. | Product/Architecture | Open |
