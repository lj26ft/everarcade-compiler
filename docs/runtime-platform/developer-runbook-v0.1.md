# EverArcade v0.1 Developer Runbook

## Purpose
Onboard external developers to the v0.1 release candidate using stable portal, package, deployment, settlement, and replay workflows.

## Onboarding Flow
1. Read the release candidate documentation.
2. Use the portal onboarding section.
3. Package a deterministic runtime artifact.
4. Deploy to the public testnet.
5. Verify settlement, GPU marketplace, and replay records.
6. Capture failure evidence before requesting support.

## PASS Criteria
- Developer can follow portal layout without unpublished steps.
- Package records appear in deployment and replay evidence.
- Testnet readiness remains `PASS`.

## FAIL Criteria
- Onboarding requires a new runtime feature.
- Package execution cannot be replayed.
- Settlement or GPU marketplace records are missing.
