# Worlds

Worlds are the primary public surface of EverArcade. A world is something you can play, verify, contribute to, operate, and fork.

## What a world profile should show

- World name and description.
- `world.evr` identifier.
- Operator and governance model.
- Proof status for replay, restore, migration, and registry publication.
- Installed capabilities.
- Treasury health.
- Contributor opportunities.
- Actions: **Play**, **Verify**, **Contribute**, and **Fork**.

## Proof status language

Use user-facing labels first: Verified, Fixture-witnessed, Implemented, Blocked, or Planned. Implementation details such as hashes, roots, journals, and accumulators belong behind **Verify**.

## Browse Worlds

The v1 world portal renders static registry fixtures from the Creator SDK while the live registry API is being connected.
