# Registry

The Registry is the public discovery layer for verified worlds.

A registry record should answer: what is this world, who operates it, what proofs exist, what capabilities are installed, how is the treasury governed, and how can someone play or contribute?

## Status model

- **Independently verified:** reproduced by a verifier from published evidence.
- **Fixture-witnessed:** reproducible over supplied fixtures.
- **Implemented:** code or CLI exists, but a test or dependency blocks full validation.
- **Planned:** public concept exists; implementation is not ready.

## Actions

- **Browse Worlds** in the portal.
- **Verify a World** before operating or forking.
- **Contribute** by improving proof coverage, metadata, capabilities, or docs.
