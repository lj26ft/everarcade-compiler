# Trusted Attester Registry Roadmap

- L1 pinned trusted key: verifier receives an out-of-band public key and rejects self-authenticated attestations.
- L2 attester registry: named attesters, key continuity, expiry, rotation, and revocation metadata.
- L3 XRPL/Xahau anchored identity: registry checkpoints and attester identity commitments anchored on-ledger.
- L4 Xaman signing UX: wallet-mediated review and signing workflows for human operators.

These layers are roadmap items, not V0.1 RC1 claims.
