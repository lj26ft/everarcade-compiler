# Creator Experience Report

## Coverage
- New Project, template selection, visual world building, local runtime launch, replay inspection, and publish are modeled as one creator workflow.
- Drag-and-drop targets cover world, hierarchy, packages, and entities while preserving deterministic asset lineage.
- Marketplace shelves expose published games, packages, templates, assets, and examples.

## Product Goal
A solo creator can create, run, publish, and deploy a game from Studio without using runtime documentation or CLI deployment steps.
