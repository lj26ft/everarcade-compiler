# Public Narrative Audit

EverArcade public messaging now leads with **Build Sovereign Worlds** and explains technical capability through world continuity, ownership, operators, developers, and players.

## Messaging shift

| Engineering-first phrase | Public narrative phrase | Use in public pages |
| --- | --- | --- |
| Deterministic Replay | Verifiable World History | Explain that world history can be checked. |
| State Lineage | World Continuity | Explain that worlds remember how they changed. |
| Execution Equivalence | Independent Verification | Explain that more than one participant can confirm what happened. |
| Runtime Package | Portable World Package | Explain that a world can move between hosts and tools. |
| Federation Runtime | Operator Federation | Explain shared stewardship and recovery. |
| Receipt Propagation | Published World Records | Explain public evidence without protocol jargon. |

## Public voice rules

- Lead with worlds, not infrastructure.
- Explain continuity as living history.
- Describe operators as stewards, not just nodes.
- Describe developers as world builders, not only protocol integrators.
- Describe players as community participants whose actions become history.
- Link to `/docs` for implementation details instead of duplicating reference documentation.

## Pages updated or added

- Homepage now uses the hero: “Build Worlds. Not Just Games.”
- `/worlds` introduces worlds without technical jargon.
- `/continuity-engine` translates continuity into living world history.
- `/developers`, `/operators`, and `/players` provide stakeholder journeys.
- `/founding-developers`, `/roadmap`, and `/community` support launch onboarding.
