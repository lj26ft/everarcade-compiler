# Arena Vanguard Launch Candidate

## Creator Workflow
Arena Vanguard content is packaged as canonical runtime, world, and deployment artifacts.

## Runtime Workflow
The runtime starts from `arena-vanguard-runtime.tar.gz`, reads deterministic manifests, and preserves replay/checkpoint continuity.

## Deployment Workflow
Operators deploy from `arena-vanguard-deployment.tar.gz` and verify package, deployment, replay continuity, and checkpoint continuity.

## Operator Workflow
Install, bootstrap, deploy, recover, upgrade, and rollback workflows are documented in `docs/operators/`.

## Recovery Workflow
Runtime crash, node restart, checkpoint restore, and world restore preserve the same replay root, world root, and continuity root.

## Multi-Node Federation Load Workflow
Four TCP EverNode endpoints join, synchronize replay/checkpoint/continuity roots, and validate deterministic per-node load before launch handoff.
- deployment artifacts: generated by `scripts/build_evernode_packages.sh`
- source control policy: generated `.tar.gz` and `.sig` files are not source controlled
- manifest-driven packages: runtime, world, deployment
- certification gate: validated
- launch candidate: validated
